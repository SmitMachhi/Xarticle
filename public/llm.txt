# Xarticle.co

Xarticle.co converts one public X URL into a structured, exportable article view.

## Product Snapshot

- Website: https://xarticle.co
- Type: Web app + stateless extraction worker
- Primary use case: read/share/archive public X long-form content in stable formats

## Supported Inputs

- Public status URLs: `https://x.com/<handle>/status/<id>`
- Public article URLs: `https://x.com/i/articles/<id>`

## Unsupported Inputs

- Private or locked account content
- Non-X domains
- Invalid URL formats

## Extraction Contract

- Endpoint: `POST /api/extract`
- Health check: `GET /health`
- Status URLs return normalized status payloads used by the frontend parser.
- Article URLs return fetched HTML for client-side parsing.

## Output Formats

- PDF (paper size + margin options)
- Markdown (`.md`)
- Offline Markdown ZIP (`article.md` + `assets/`) when media is present

## Runtime Notes

- Backend is stateless.
- Exports are generated in the browser session.
- Parsing reliability depends on upstream public X payload/markup stability.
- Some metrics may be missing when upstream data is unavailable.
